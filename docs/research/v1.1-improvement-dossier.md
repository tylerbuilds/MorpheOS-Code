# MorpheOS Code v1.1 — Improvement Dossier

> Research compiled from 6 agents analyzing 25+ CLI coding tools, TUI patterns, and agent-native architectures.
> Date: 2026-07-20

## Executive Summary

MorpheOS Code v1.0 is a solid foundation: TUI, personality, safety gate, agent-native JSON mode, MCP tool. To compete with tools at 47K–138K stars, v1.1 needs three pillars:

1. **Codebase understanding** — repo maps, context providers, semantic indexing
2. **Agent orchestration** — subagents, permission granularity, checkpoint/rewind
3. **Distribution breadth** — MCP everywhere, ACP compatibility, editor integration

---

## Tier 1: Differentiators (steal these first)

### 1. Repository Map with Graph-Ranking *(from Aider)*
**Impact: Critical | Effort: High**

Aider's signature feature. Builds a symbol-level dependency graph of the entire codebase (classes, functions, signatures) using tree-sitter, then uses graph-ranking to select the most relevant subgraph that fits within the token budget.

**For v1.1:**
- Tree-sitter parser integration for TypeScript, Python, Rust, Go, etc.
- Auto-build repo map on session start (cached by content hash, cross-branch deduplication)
- Token-budget-aware ranking: send most relevant symbols to the LLM
- Repository map accessible via tool: `morpheos --map "auth module"` shows dependency graph
- This is the single highest-impact feature — it solves "the model doesn't know about my codebase"

### 2. Parameter-Level Permission Rules *(from Claude Code)*
**Impact: Critical | Effort: Medium**

Replace binary allow/deny with pattern-matched permission rules. Claude Code's killer feature: `Bash(git push)` (ask) vs `Bash(git *)` (allow) vs `WebFetch(https://api.mycompany.com/*)` (allow).

**For v1.1:**
- Tool, parameter, and domain-level matchers with wildcards and globs
- Permission modes: Manual → Plan → Auto (cycleable with a hotkey)
- Plan mode shows a markdown document of what the agent will do before execution
- Rules persist per-session and per-project in config files

### 3. Background-by-Default for Long Operations *(from Claude Code)*
**Impact: High | Effort: Medium**

Sub-agents, shell commands, and MCP calls auto-background when they exceed a timeout. The main session never freezes. A dashboard shows running background work.

**For v1.1:**
- `morpheos --bg "run the full test suite"` — returns session ID immediately
- `morpheos agents` — dashboard of running/finished background work
- `morpheos attach <id>` — rejoin a background session
- Timeout threshold auto-moves stalled operations to background

### 4. Checkpoint/Rewind as Conversation Primitive *(from Cursor & Claude Code)*
**Impact: High | Effort: Medium**

Before every significant file edit, take a local snapshot (not git-dependent). Any message can be a fork point. Restore code state, keep history, or both.

**For v1.1:**
- Auto-checkpoint before file mutations
- `/rewind` command to restore to any previous message
- `/fork` to explore an alternative without losing current state
- Checkpoints stored in `.morpheos/checkpoints/` (gitignored by default)

### 5. Adversary Mode *(from Goose)*
**Impact: High | Effort: Low-Medium**

A silent, independent agent reviewer that inspects every tool call before execution against user-written natural language policies. Contextually understands intent, not just pattern-matching.

**For v1.1:**
- `morpheos --adversary "Never delete files outside the src/ directory"` 
- Second LLM call runs in parallel with the main agent
- Blocks or warns on policy violations
- Policies are natural language, stored in `.morpheos/policies.md`
- This is categorically superior to regex-based command blocking

## Tier 2: Quality of Life (high impact, moderate effort)

### 6. MCP Everywhere — Actually Do It Right *(from community demand)*
**Impact: High | Effort: Medium**

Both Aider (47K stars) and Open Interpreter (67K stars) have MCP as their #1 community request. Neither has fully delivered. This is the single highest-impact differentiator available.

**For v1.1:**
- MCP server consumption: `morpheos mcp add github-tools` — connects to any MCP server
- MCP server exposure: MorpheOS Code itself as an MCP server (already partially done with `morpheos_chat`)
- Streamable HTTP transport for remote access
- Tool discovery, output schemas, structured validation
- MCP marketplace browsable from within the TUI (`:marketplace`)

### 7. Auto Memory That Writes Itself *(from Claude Code)*
**Impact: Medium | Effort: Low**

The agent writes its own learnings (build commands, debugging workflows, preferences) to a MEMORY.md across sessions. First 200 lines / 25KB load at session start.

**For v1.1:**
- `.morpheos/MEMORY.md` — auto-maintained by the agent
- Topic files load on demand: `debugging.md`, `api-conventions.md`, `testing.md`
- Agent proposes memory writes; user can `/accept-memory` or `/reject-memory`
- Memory index loaded at session start for fast context

### 8. Architect/Editor Model Pairing *(from Aider)*
**Impact: Medium | Effort: Low**

Split reasoning from editing across two models: strong reasoner (Pro) plans, cheap editor (Flash) produces edits. Cuts costs 40-60% on complex tasks.

**For v1.1:**
- `morpheos --architect pro --editor flash` — model pairing
- Architect writes a plan, editor executes it
- Default: Flash for simple edits, Pro only for architecture/planning
- Already have the model routing infrastructure from v1.0

### 9. CLI-Accessible Skills & Plugins *(from Claude Code, Goose)*
**Impact: Medium | Effort: Medium**

A single mechanism for both simple slash commands and advanced skills. `.morpheos/skills/` directory with SKILL.md files. Plugin system for distributing reusable bundles.

**For v1.1:**
- `/skill deploy` — runs `.morpheos/skills/deploy/SKILL.md`
- Skills can bundle hooks, tools, and sub-agents
- Plugin marketplace: `morpheos plugin install github.com/user/morpheos-deploy`
- Project-level and user-level plugin directories

### 10. Universal `/` Filter + Command Palette *(from lazygit, k9s, fzf)*
**Impact: Medium | Effort: Low**

Every list view is filterable. A `:` command palette makes every action discoverable with fuzzy matching.

**For v1.1:**
- `/` in any list: files, commits, sessions, test failures
- `:` command palette: fuzzy-match all available actions
- Express search syntax: `^prefix`, `'exact`, `!inverse`, `|` OR

## Tier 3: Maturity Signals (medium impact, higher effort)

### 11. Hooks at Lifecycle Points *(from Claude Code, Goose)*
**Impact: Medium | Effort: Medium**

Shell commands or HTTP calls at PreToolUse, PostToolUse, SessionStart, SessionEnd, PreCompact, UserPromptSubmit.

**For v1.1:**
- `.morpheos/hooks/pre-tool-use.sh` — runs before every tool call
- Can block, allow, modify permissions, inject context
- JSON on stdin/stdout for structured decisions
- Built-in hooks: auto-format on save, auto-lint on edit, auto-test on build

### 12. Usage Attribution by Component *(from Claude Code)*
**Impact: Medium | Effort: Low**

Show where token budget went: skills, sub-agents, MCP servers, plugins. Flag unusually expensive sessions.

**For v1.1:**
- `/cost breakdown` — shows per-component token usage
- Sessions flagged when >2x average cost
- Model recommendation: "This task would cost £0.02 on Flash vs £0.12 on Pro"

### 13. Inline Diff Review with Per-Edit Accept/Reject *(from Copilot)*
**Impact: Medium | Effort: High**

Show proposed changes as side-by-side diffs with individual accept/discard. Most important IDE-agent UX pattern.

**For v1.1:**
- After agent makes changes: show diff with accept/reject per hunk
- `y` accept hunk, `n` reject, `a` accept all, `q` reject all
- Undo stack navigable with `z`/`Shift+Z`

### 14. Sub-Agent Orchestration *(from Claude Code, Codex Ultra)*
**Impact: Medium | Effort: High**

Spawn parallel sub-agents for independent tasks. Background execution with structured results.

**For v1.1:**
- `morpheos --agents 3 "refactor all three services"` — parallel sub-agents
- Sub-agents get isolated context, restricted tools, custom prompts
- Results merge into main session
- Sibling roster for inter-agent communication

### 15. ACP Compatibility *(from Goose, Open Interpreter)*
**Impact: Medium | Effort: Medium**

Agent Communication Protocol. Makes MorpheOS Code pluggable into any editor or agent framework that speaks ACP.

**For v1.1:**
- ACP server mode: `morpheos --acp` or `morpheos serve --acp`
- ACP client mode: use other agents as providers
- This makes MorpheOS a "meta-agent" — orchestrating other coding agents

## Architecture: What the research tells us

### The next generation of coding agents has a common shape:

```
┌─────────────────────────────────────────────┐
│              Command Intake                  │
│  CLI --json  │  MCP server  │  ACP server   │
├─────────────────────────────────────────────┤
│           Agent Orchestrator                 │
│  Sub-agents  │  Background  │  Parallel     │
│  Permission rules  │  Adversary reviewer    │
├─────────────────────────────────────────────┤
│           Codebase Understanding             │
│  Repo map  │  Context providers  │  Index   │
│  Auto memory  │  Project config              │
├─────────────────────────────────────────────┤
│           Tool Execution                     │
│  Built-in tools  │  MCP tools  │  Plugins   │
│  Sandbox  │  Checkpoint  │  Undo            │
├─────────────────────────────────────────────┤
│           State & Persistence                │
│  Sessions  │  Memory  │  Audit trail        │
│  Cost ledger  │  Usage attribution          │
└─────────────────────────────────────────────┘
```

MorpheOS Code v1.0 has the bottom two layers solidly built (tool execution + state). v1.1 should focus on the top two layers (intake breadth + orchestration) and the middle layer (codebase understanding).

### What NOT to build (anti-patterns from research)

- ❌ Single-provider lock-in (Codex's biggest criticism)
- ❌ Silently accept invalid config JSON (Claude Code pain point)
- ❌ Remove features without migration paths (Cursor's `/undo` removal)
- ❌ Let model updates silently change agent behavior
- ❌ Gate core UX behind subscription tiers
- ❌ Require Electron/GPU for a terminal tool
